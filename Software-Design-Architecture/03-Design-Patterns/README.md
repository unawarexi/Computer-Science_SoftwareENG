# 03 - Design Patterns

Explore the most important design patterns every architect should know. Learn how to recognize, implement, and adapt patterns to solve common software problems.

## What You'll Learn

- Creational, Structural, and Behavioral patterns
- When to use each pattern
- Anti-patterns and refactoring

## Weekly Breakdown

**Week 1: Creational Patterns**
- Singleton, Factory, Abstract Factory, Builder, Prototype

**Week 2: Structural Patterns**
- Adapter, Facade, Proxy, Bridge, Composite, Decorator

**Week 3: Behavioral Patterns**
- Observer, Strategy, State, Command, Chain of Responsibility

**Week 4: Anti-Patterns & Refactoring**
- Recognizing and avoiding anti-patterns
- Refactoring code for better design

---

By the end, you'll be able to apply patterns to real-world problems with confidence.
