# Preemption and Context Switching

## Preemption
- The ability of the OS to interrupt a running process and assign the CPU to another process.
- Enables responsive multitasking and prevents any single process from monopolizing the CPU.

## Context Switching
- The process of saving the state of a running process and loading the state of the next scheduled process.
- Involves saving registers, program counter, and other process-specific data.
- Context switching introduces overhead but is essential for multitasking.
