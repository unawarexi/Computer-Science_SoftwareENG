# Thrashing and Swapping

## Thrashing
- Occurs when a system spends more time swapping pages in and out of memory than executing processes.
- Caused by insufficient memory or poor page replacement policies.
- Leads to severe performance degradation.

## Swapping
- Moving entire processes between main memory and secondary storage.
- Used to free up memory for other processes.
- Swapping is less common in modern systems due to virtual memory.
